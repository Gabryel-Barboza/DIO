Hierarquia do Efeito Cascata

Estabelece uma prioridade para a aplicação de uma regra de CSS

Se houver um conflito, o navegador irá:

  1. Analisar todas as regras;
  2. Ver o nível de importância do elemento;
  3. Comparar a especificidade das declarações de mesmo;nível (cálculo de especificidade);
  4. Em caso de empate de grau de especificidade, irá selecionar o último seletor declarado.

Texto 02 - ID e classe

Texto 03 - inline

Cálculo de Especificidade

  1. inline - 1000 pontos
  2. ID - 100 pontos
  3. classes, pseudo-classes e atributos - 10 pontos
  4. Tag HTML - 1 ponto
  • #exemplo h1.titulo - 111 pontos
  • a:hover - 11 pontos
  • div p - 2 pontos

Para a pseudo-classe :not não se aplica o cálculo de especificidade

!Important

Nivel de prioridade do efeito em cascata:

  1. Declarações com !important do usuário (devtools)
  2. Declarações com !important do desenvolvedor
  3. Folha de estilo do desenvolvedor
    1. CSS inline
    2. CSS interno
    3. CSS externo
    4. Folhas de estilo padrão navegador
